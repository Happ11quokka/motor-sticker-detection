# 최근 20건 쇼케이스 큐레이션 — 설계 (Design Spec)

> 작성일: 2026-06-02 · 브랜치: `feat/showcase-recent20`
> 대상: 데모 대시보드의 "최근 20건" 결과 패널 (`student_template/app.py`)

## 1. 목적 / 문제

발표 시연용 대시보드의 "최근 20건" 표가 현재 **거의 전부 초록(정상)** 한 종류뿐이라 단조롭다.
이 패널을 **다양한 불량 등급이 섞인 큐레이션 쇼케이스**로 바꿔, 한 화면에서 시스템의 분류
능력과 견고성(흑백·블러에도 검출)을 보여준다.

## 2. 확정 요구사항

- 등급 4종을 섞어 보여준다: **정상 · 경미한 불량 · 심각한 불량 · 관련없음(신규 태그)**.
- 정상/경미한 불량/심각한 불량 행은 스티커 칸의 **`O` 토글**을 누르면 그 그룹의
  **남은 사진 2장(파일 순서)** 을 함께 펼쳐 보여준다. **관련없음 행은 토글 없음.**
- 20건은 **시간순이 아니라 임의로 큐레이션 배치**한다("최근"이라는 제목은 유지).
- 20건 안에 **흑백(grayscale) · 블러(blurred)로 분석된 이미지도 섞는다** (별도 배지 없음).
- `20240817_000743.jpg`(번호 112, 초록) 를 **초록 대표로 반드시 포함**(맨 앞 권장).

## 3. 확정 결정사항

| 항목 | 결정 | 근거 |
|---|---|---|
| 적용 범위 | **데모 전용 큐레이션** | 실제 업로드/GPT 분석 로직은 불변 — 발표 안전 |
| 심각한 불량 표현 | **라벨만**(빨강 이미지 제작 ❌) | 데이터에 빨강 스티커 없음, 사용자 확정 |
| 색-라벨 충돌 회피 | 경미/심각 재지정 행은 **흑백·블러 이미지** 사용 | 무채색이라 "초록 썸네일 vs 빨강 배지" 모순 없음 + 흑백·블러 포함 요구 동시 충족 |
| 흑백/블러 표식 | **별도 배지 없음**, 그냥 섞음 | 사용자 확정 |
| 토글 구현 | **CSS-only**(JS 미사용) | Gradio가 JS 제거. 기존 라이트박스 `:target` 방식이 이미 작동 |

## 4. 핵심 데이터 구조 (이미 존재)

`student_template/data/seed/results.json` 의 `groups[]` 는 각 그룹이 **사진 3장**(`images[]`)을
담고, 그중 1장이 `has_sticker: true`. → **"남은 사진 2장" = 해당 그룹 `images` 에서 스티커
이미지를 뺀 나머지 2장(원래 파일 순서 유지).** 토글은 이 구조를 그대로 조회해 사용한다.

이미지 자산:
- 컬러 원본: `data/motor_checker_2/*.jpg` (초록 다수 + 노랑 1: 번호 169)
- 흑백: `data/motor_checker_2/grayscale/*.jpg` (36장)
- 블러: `data/motor_checker_2/blurred/*`
- 관련없음(비-모터 엣지케이스): `test_img/` 내 모니터 화면·잡 썸네일류
  (현 시드 그룹 16~18: `*thumb_d_*`, `c92257_99`, `dmbt6_*`, `SMNM*`, `abd64b*` 등)

## 5. 설계

### 5.1 큐레이션된 20행 구성

분포(목표): **정상 ~9 · 경미한 불량 ~3 · 심각한 불량 ~2–3 · 관련없음 ~5** (합 20)
- 정상: 실제 초록 이미지 (112/`000743` 포함, 맨 앞)
- 경미한 불량: 실제 노랑(169) 1 + 흑백 재지정 약 2
- 심각한 불량: 흑백·블러 재지정 (라벨만, 빨강 배지로 "심각" 신호)
- 관련없음: 실제 비-모터 엣지케이스 (스티커 X, 토글 없음)
- 흑백 ~5 · 블러 ~3 행을 위 등급들에 분산 (배지 없음)
- **순서**: 등급이 교차하도록 배치(정상-관련없음-경미-심각-정상…). 1번 = 초록 대표.

각 스티커 행은 3장 그룹을 보유 → 토글 시 남은 2장(같은 변형: 컬러/흑백/블러로 일관) 표시.

### 5.2 렌더러 변경 — `app.py`

- `render_results_html()`
  - 스티커 칸: 정상/경미/심각 행 → **`O` 토글**(클릭 시 남은 2장 펼침, 재클릭/타행 클릭 시 접힘).
    관련없음 행 → `X`, 토글 없음.
  - 펼침 영역: 남은 2장 썸네일 + 파일명(파일 순서). 라이트박스(확대)는 기존대로 유지.
  - 배지: `관련없음`용 중립 회색 배지 `.b-irrelevant` 추가. 기존 정상/경미/심각 배지 유지.
- 데이터 공급(`get_dashboard_data()` 또는 신규 `get_showcase_rows()`)
  - 큐레이션 20행을 **큐레이션 순서대로** 반환(`has_sticker` 필터 ❌, 시간 역순 ❌).
  - 각 행에 **남은 2장 파일명**을 동봉(소속 그룹에서 조회).
- 통계 카드(정상/경미/심각 분포)는 큐레이션 데이터로 자동 재계산 — 별도 손대지 않음.

### 5.3 빌더 스크립트 — `teacher_tools/build_showcase.py` (신규)

`teacher_tools/preseed.py` 관례를 따라 **결정적으로** 시드를 조립:
1. 컬러/흑백/블러/관련없음 후보에서 이미지 선택(112/000743·169 고정 포함).
2. 타임스탬프명으로 `data/uploads`(로컬) + `data/seed/uploads`(번들)에 복사(리사이즈).
3. 그룹(3장)·라벨(재지정 포함)·큐레이션 순서를 구성해 `results.json` + `seed/results.json` 기록.
4. 재실행 가능(idempotent) — 발표 전 분포/순서 조정 용이.

### 5.4 토글 메커니즘 (CSS-only)

1순위 `<details>/<summary>`(다중 펼침 가능), Gradio 살균에 막히면 기존 검증된
`:target` 방식(한 번에 하나 펼침)으로 폴백. **구현 시 실제 Gradio HTML 컴포넌트에서
렌더/작동을 반드시 눈으로 검증**(스크린샷).

## 6. 손대는 파일

| 파일 | 변경 |
|---|---|
| `student_template/app.py` | 렌더러(토글·관련없음 배지) + 데이터 공급(큐레이션 순서) |
| `teacher_tools/build_showcase.py` | **신규** 빌더 스크립트 |
| `student_template/data/seed/results.json`, `seed/uploads/*` | 재생성(큐레이션 번들) |
| `student_template/data/results.json`, `uploads/*` | 재생성(로컬 확인용) |

## 7. 가정

1. 심각한 불량 = 흑백/블러 이미지에 라벨만(빨강 배지 유지).
2. 관련없음 행은 토글 없음(보여줄 그룹 사진 없음).
3. 통계 카드 구성은 그대로(데이터만 재계산).
4. 표 제목 "최근 20건" 문구 유지.

## 8. 검증 기준 (완료 정의)

- [ ] 대시보드 "최근 20건"에 4종 등급이 섞여 표시(정상/경미/심각/관련없음).
- [ ] 정상/경미/심각 행의 `O` 토글 클릭 → 남은 2장(파일 순서) 펼쳐짐. 관련없음은 토글 없음.
- [ ] 20건에 흑백·블러 이미지가 포함(배지 없음).
- [ ] `20240817_000743.jpg`(번호 112, 초록) 포함, 1번 자리.
- [ ] 실제 업로드/분석 경로 미회귀(앱 정상 기동, 라이트박스 동작 유지).
- [ ] Gradio 화면 스크린샷으로 토글·배지 육안 확인.
